# 演示脚本 · 三条主线

把这份给评审人。每条主线 5–8 分钟，围绕 PRD V1.0.0 的金链条走，三条线之间可以独立看。

---

## 路线 A · C 端读者视角（前台 H5 · 8 分钟）

> 模拟一位医生扫到《心血管分册》后封底二维码后的全流程。

**准备**：浏览器开 <http://localhost:5173>，DevTools 设 375×812 视口（或在桌面看居中手机外框版即可）。
**打开演示控制台**：右下角橙色按钮，确认当前角色是「免费用户」、机构是「中国医学临床百家」。

| 步骤 | 操作 | 看点 |
|---|---|---|
| 1 | 直接到 `/`（落地页） | Agent 形象 + 渐变标题 +「不限次数 / 章节溯源 / 图音视频 / 永享买断」四个特性卡 |
| 2 | 点「微信授权登录」 | 1.5s 假回调（系统级原型态）→ 自动跳手机号绑定 |
| 3 | 手机号 138xxxx1234 + 验证码 1234 | 60s 冷却倒计时；机构内手机号唯一文案；`E_AUTH_002` 错误码联动 |
| 4 | 进入 AI 会话页 | 顶栏 Agent / 客服入口 / 我的；欢迎语区含三条入门追问 |
| 5 | 点「冠脉造影术前抗凝怎么管？」 | **流式打字机首字 ≤ 500ms 出现**；流完后下方并列三件套 |
| 6 | 仔细看答案 | 三件套：① 多模态卡片（免费图 / 会员视频锁 / 永享音频锁 ¥168） ② 回答溯源（带页码、时间戳）③ 推荐追问 + 答案末延伸视图 |
| 7 | 点「永享 ¥168 音频」锁标 | 弹起付费墙底抽屉：会员 ¥19/月 ⊕ 永享 ¥168 + 兑换码 Tab |
| 8 | 选「永享买断」 | 跳支付过场（控制台可切「成功 / 失败 / 取消」）→ 支付成功页 → 返回会话 |
| 9 | 回会话再点同一卡 | **不再有锁标**（permanent_grants 已写入）—— 验证权益生效 |
| 10 | 点顶栏头像进「我的」 | 用户卡片显示「会员 + 永享」；KPI 卡 + 入口列表（会员中心 / 永享 / 订单 / 历史 / 客服） |
| 11 | 进「历史会话」 | 看到刚才的会话，点进详情可回看完整答案 |
| 12 | 顶栏「···」打开客服弹窗 | 企微/个微二维码 + 电话 + 邮箱（来自机构后台「客服配置」） |

**演示控制台切换演示**：
- 切到「AI 会员」→ 回会话点「会员视频锁」→ **直出可看**
- 切到「未登录」→ 落地页重新走一遍
- 切机构「财经出版社」→ 看欢迎语 / 客服信息切换（强隔离）

## 路线 B · 机构管理员视角（机构后台 · 6 分钟）

> 模拟出版社编辑要把一本新书做成 KP 上线的全流程。

**准备**：浏览器开 <http://localhost:5174/login>。
**演示账号**：`admin@org_med` / `demo1234` / 图形验证码 `WJ4X`。

| 步骤 | 操作 | 看点 |
|---|---|---|
| 1 | 登录 → 主控台 | 4 KPI 卡（GMV / 订单 / 提问 / 活跃 KP）+ 14 天提问趋势 + 8.9% 付费转化 + 热门 KP 表 |
| 2 | 点侧栏「知识产品 KP」 | 列表 6 个 KP（含一个「草稿」状态）+ 表格筛选 + 状态色标签（free / member / forever 视觉差异） |
| 3 | 点「新建 KP」按钮 | 弹窗：名称 + 5 种颗粒度（单本书 / 系列 / 专家 / 领域 / 自定义） |
| 4 | 进「心血管分册」详情 | 4 个 Tab：基本信息 / 文件与切片 / 二维码 / 定价与权益 |
| 5 | Tab「文件与切片」 | 3 个文件状态（就绪 / 解析中）+ 切片预览前 300 字（PRD 5.3.3） |
| 6 | 点「上传文件」 | 弹窗：拖拽区 + 支持格式 + 异步任务进度抽屉演示 |
| 7 | Tab「二维码」 | 3 个码包卡（已分发 5 万 / 1 万 / 草稿 20 万）+ 扫码漏斗 |
| 8 | 点「生成新码包」 | 弹窗：码量、PNG/SVG/PDF、绑定权益（不绑/会员/永享）|
| 9 | Tab「定价与权益」 | **免费 ⇄ 会员 互斥** + **永享独立可叠加** + 永享单价 + 历史价格记录 |
| 10 | 侧栏「Agent 人设」 | 形象 / 头像上传 / GIF / TTS 参考音 / 大模型选择（4 档：平台默认 / 通义 / 豆包 / OpenAI）/ 富文本欢迎语 + 入门追问 |
| 11 | 顶部「打开前台 H5」 | 跳到 5173 看 C 端实际渲染效果 |

## 路线 C · 平台超管视角（平台超管 · 5 分钟）

> 模拟平台运营给新机构开 LLM/支付参数的全流程。

**准备**：浏览器开 <http://localhost:5175/login>。
**演示账号**：`superadmin` / `demo1234` / 2FA 动态码 `123456`。

| 步骤 | 操作 | 看点 |
|---|---|---|
| 1 | 登录 → 2FA 弹窗 | 6 位动态码 + 输入 123456 通过 |
| 2 | 主控台 | 4 KPI（入驻机构 / GMV / token / 告警）+ 14 天 LLM token 趋势区域图 + 各机构用量表 |
| 3 | 侧栏「机构管理」 | 3 家机构表 + 状态 / token 用量进度条 / LLM 配置 |
| 4 | 点「新增机构」 | 弹窗：机构名 + 编号（兑换码前缀） |
| 5 | 进「中国医学临床百家」详情 | 4 Tab：基本资料 / LLM&联网 / 微信支付 / 用量看板 |
| 6 | Tab「LLM&联网」 | 平台默认 vs 自配厂商二选一 → 月配额 5000 万 token + 80%/100% 告警阈值 |
| 7 | Tab「微信支付」 | 商户号 / API 证书上传 / V3 密钥（KEY 末四位明文）+ 资金 100% 进机构账户声明 |
| 8 | Tab「用量看板」 | 4 KPI（已用 / 上限 / 使用率 / 预计超额）+ 7 天用量柱 |
| 9 | 侧栏「默认 LLM 用量」 | 全平台 token 看板 + 7 天堆叠柱（按机构） + 三家机构明细表（含告警状态列） |

**安全演示**：
- 点「删除机构」按钮 → toast「需要超管密码 + 留存数据声明」（高危拦截）
- 所有写操作（修改 KEY、修改配额）会落审计日志

## 演示彩蛋

- 三家示例机构（医学/财经/文学）的客服信息、品牌色、KP 列表都有差异，可以演示「机构数据强隔离」的 PRD 铁律
- 控制台 / Network 面板里能看到所有 `[track] xxx` 埋点，对应 PRD 第 6 章事件清单
- 所有用户角色 / 订单 / 会话状态都存在 localStorage，关掉浏览器再开仍在；想清空就点演示控制台「重置」
