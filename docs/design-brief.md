## 一、母 Prompt（默认填的是 H5，跑 PC 时替换【本次范围】与「关键界面」两段）

```
你是 AI 问书的主视觉设计师。请为产品产出高保真界面 + 一份可交付给开发的 design.md。

【本次范围】机构前台 H5（移动端 375×812）
（第二次跑时把这一行替换为：机构后台 Web PC + 平台超管 Web PC，PC 内容区按 1440 宽设计）

== 产品是什么 ==
AI 问书：出版方/机构把纸书内容做成「可对话的 AI 版作者」。读者扫书上的二维码进入 H5，与绑定该书的 Agent 对话，答案逐字流式输出并附带原书图/音/视多模态内容与溯源；受版权保护的内容按会员/永享权益解锁。机构在后台管理知识产品(KP)、Agent、定价、数据；平台超管管理所有机构。本期是高保真原型（只 mock，不接真后端）。

== 视觉方向：方向 B（已锁定，请严格沿用，不要另起配色）==
DNA：奶白底 + 电光靛 + 朝霞橙，新潮、友好、有「当代 AI 工具」质感（参照豆包/Kimi 的语言），中文阅读舒适。
颜色：
- 背景 bg #f6f4f0 / 卡片 surface #ffffff / 次级面 surface2 #fbf9f4
- 墨蓝主文 ink #14182a / 次文 ink2 #2c3046 / 弱文 muted #797d8c / 更弱 subtle #c1c3cc
- 分隔线 hairline rgba(20,24,42,0.10)
- 主色 电光靛 indigo #4f46e5 / 深 #3730a3 / 浅底 indigoSoft #e7e8ff
- 强调 朝霞橙 coral #ff7a5c / 浅底 coralSoft #ffe4dd
- 语义：success #1f9d70 / warning #d97706 / danger #d92d20
字体：正文 "Noto Sans SC"；大标题/大数字 "Manrope"；数字/代码 "DM Mono"。
圆角阶梯 6/8/12/14/16/20/24/full；间距 4 的倍数；阴影含一档 glow `0 6px 28px rgba(79,70,229,.28)`。
权益标签语义：免费=中性灰、会员=靛蓝(indigoSoft 底)、永享=朝霞橙(coralSoft 底)。

== 卡片与图片填充规则（硬约束）==
- 后台列表一律以卡片为主：
  · Agent 列表卡片 = Agent 形象图 + 关键字段（名称、类型[机构/普通]、关联 KP 数、声音/TTS 状态）
  · KP 列表卡片 = 9:16 封面图 + 关键字段（名称、来源标签[自建/共享]、状态、关联 Agent、文件数、提问数）
- 卡片图片优先 cover 铺满、不留白；当为保持比例不得不留白时，留白区必须做视觉处理：
  用同一张图高斯模糊(毛玻璃)铺底 + 前景原图居中，或用主题色(靛/橙)柔性渐变填充。
  禁止出现生硬的纯白/纯灰边。图片四角与卡片圆角一致。

== UX 原则（硬约束）==
- 用户体验优先级高于技术与代码整洁。不让用户思考、系统承担复杂性、渐进式展示、反馈引导下一步行动。
- 移动端遵守安全区与单手可达；PC 端左侧菜单 + 顶栏 + 内容区栅格，信息密度专业但不拥挤。
- 全中文界面。

== 需要设计的关键界面（本次范围内挑高信息密度的逐屏做高保真）==
[机构前台 H5]
1. 登录：落地页＝AI 问书 icon + slogan + 两个登录按钮「微信登录 / 手机号登录」（无问候语）；微信登录成功后若未绑手机号则引导绑定（含「暂不绑定」），手机号登录＝手机号 + 验证码直接登录；手机号在本机构已存在时走本人验证合并
2. AI 会话主页（布局沿用现有代码）：顶栏＝左 ☰（打开「我的」左侧抽屉，抽屉底部为「我的」入口、上方为历史会话）/ 居中标题「AI 问书」+ 副标「内容由 AI 生成」/ 右上 📞实时电话 + ➕新建会话；新增 sticky Agent 形象（下滚渐隐、上滚复原）；slogan「有问题，就问 AI 问书」（"问AI问书"靛→橙渐变）+ 副标语「答案有出处 · 知识更可信」+ 其下 3 条入门示例问题（右侧 ✦ 图标，系统按 KP 生成，点击即发）；用户气泡 + AI 答案打字机流式；多模态卡片＝**文字区在上、媒体区（图/音/视）在文字区下方**，两区视觉区分、媒体并列可左右滑、**顺序按后台返回的相关度从左到右（非固定序）**、受限内容带锁标；答案下方一行＝左侧「复制/点赞/反馈」三个操作 icon + 右侧「参考 N 篇知识」入口；推荐追问在卡片**下方上下纵向堆叠**；输入栏「发消息或按住说话」+ 下方功能 chips（深度思考 / 智能搜索）+ 语音波形按钮
3. 溯源右侧抽屉：按相关度从高到低排序的列表（**不展示相关度数值**），每条＝源头图48×48 + KP名 + 文件名 + 章节页码；条目点击：book 类型跳纸书购买链接（空则 toast「无跳转链接」3 秒消失）、web 类型直接打开来源网页
4. 多模态全屏 lightbox：顶部固定「因版权限制，暂不支持保存下载」，底部缩略图条可点切换，主区左右滑
5. 实时电话式语音页（**会员专属**，未开通则引导开通）：中央 Agent 形象；下方居中挂断、左静音、右字幕；挂断上方有「正在听…/正在回答…」状态文案+动画；字幕开启时 Agent 形象隐藏、对话文本自上而下左对齐（用户浅灰、AI 深黑）
6. AI 会员订阅页（首月折扣 + 月度两档，自动续费协议入口 + 72h 宽限期说明）+ 微信支付结果页
7. 我的：会员中心 / 我的永享（放大预览带版权提示）/ 我的订单（全部·会员·永享·兑换码 四 Tab）/ 联系客服弹窗（一个二维码 + 选填电话邮箱，空字段不留白块）

== 交付物 ==
A. 上述关键界面的高保真稿（移动端按 375×812 画；PC 端按 1440 宽画），含常态/空态/加载/受限(锁)/错误等关键状态。
B. 一份 design.md（中文），结构如下，供开发直接落地：
   1) 设计基调与品牌 DNA（一段话讲清气质 + 该传达/该避免的感觉）
   2) 设计 token 全集：颜色（语义命名）、字体与字号阶梯（H1…body…caption 的 px/字重/行高）、间距、圆角、阴影、层级 z-index
   3) 权益标签与状态色规则
   4) 组件规范（逐个给尺寸/内边距/状态/交互）：按钮(主/次/文字/危险)、输入框与验证码、卡片(含上面的图片填充规则)、标签 chip、Tab、二级 Tab、抽屉 Drawer、弹窗 Dialog、Toast、表格(后台)、列表卡片、空态、加载/骨架、锁标、付费墙卡、Agent 形象气泡、多模态卡片、溯源条
   5) 布局骨架：H5（顶栏/安全区/底部导航/sticky 规则）；PC（左侧菜单宽度+顶栏高度+内容区栅格列/间距）
   6) 关键页面逐屏说明：布局结构、信息层级、用到的组件、状态、跳转关系
   7) 动效规范：打字机流式、sticky 形象渐隐、抽屉滑入、电话页语音状态动画、字幕样式、按钮/卡片反馈
   8) 图标系统规范（重点交付，移动端 + 后台共用同一套，第二次跑 PC 沿用并扩充）：
      - 统一线性(stroke)图标系统：基于 24×24 设计网格；统一描边 1.75px@24、圆角端点与拐角(round cap/join)、视觉重量一致；圆润友好、与 Agent 形象气泡同调，避免尖锐/写实/拟物/多色乱用。
      - 尺寸阶梯：行内 16/20、导航与功能 24、强调 28+，及各尺寸下的描边换算。
      - 颜色规则：默认 ink2、弱化 muted、选中/激活 indigo、危险 danger。
      - 状态：默认 / hover / active / disabled 四态画法。
      - 推荐以成熟圆角线性库为基底（如 Lucide 或 Phosphor）统一选型，保证「不丑且风格一致」，并说明在其基础上如何对齐本产品规范。
      - 高频图标清单（逐个给语义 + 选型 + 用法）：发送、短语音、电话、静音、字幕、锁、放大、溯源/书、KP、Agent、会员、永享、兑换码、扫码/相机、上传、筛选、导出、二维码、分享、设置、机构、用户、数据看板、返回、关闭、更多。
      - 输出：图标总览清单图 + 命名规范 + Do/Don't 对比示例。
   9) 插画 / Agent 形象画法、图片占位比例（KP 封面 9:16、源头图 48×48、Agent 形象图比例）。
   10) 响应式与适配说明。

要求：design.md 里所有数值要具体可执行（给 px、给色值），不要停留在形容词。先给我整体风格定调与 1–2 个核心界面，确认方向后再铺开其余界面与完整 design.md。
```
