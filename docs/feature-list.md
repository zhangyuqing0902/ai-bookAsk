## 一、机构前台 H5（C 端用户）

> **【0613 登录与手机号绑定 · 详细规则（PRD 依据）】**
>
> **A. 微信网页授权可获取的信息**
> - 经微信网页授权（scope=`snsapi_userinfo`）可获取：`openid`、`unionid`、**昵称、头像、性别、地区（国家 / 省 / 市）**。
> - **不可获取手机号**：H5 公众号网页授权无手机号能力（微信「一键授权手机号」为小程序 `getPhoneNumber` 组件专属，H5 不具备），故本产品手机号一律由用户手动输入「手机号 + 验证码」绑定。
>
> **B. 登录流程（按打开环境分流）**
> - **B1 微信内置浏览器**：点「微信授权登录」→ 唤起微信官方授权弹窗（授权项：昵称 / 头像 / 性别 / 地区）→ 点「允许」携 `code` 回调，拉取上述资料写入账户 → 进入 AI 会话首页（手机号为空，后续按需绑定）；点「拒绝」→ 留在登录落地页。**不出二维码，且不再弹「获取手机号」授权窗（H5 不具备该能力）。**
> - **B2 非微信浏览器**：点「微信授权登录」→ 由微信域名 `open.weixin.qq.com` 渲染动态二维码 → 用户微信扫码、手机端确认「允许 / 拒绝」→「允许」携 `code` 回调，流程同 B1 → 进入首页。
> - **B3 手机号验证码登录（两环境一致）**：输入手机号 + 短信验证码，校验通过即登录。**此方式不获取、不绑定任何微信信息**（手机号登录用户无需微信授权）。
> - 落地页提供「演示环境：微信内 / 浏览器打开」切换——**仅原型演示用**，生产环境由 UserAgent（是否含 `MicroMessenger`）自动判定，无此切换。
>
> **C. 头像 / 昵称 / 性别 / 地区 的来源**
> - 微信环境：登录授权时自动带回，个人资料页只读展示并标注「来自微信」。
> - 非微信环境：个人资料页支持用户**上传头像、自行填写性别与地区**。
>
> **D. 手机号绑定（统一「手机号 + 验证码」，不分环境）**
> - 微信登录用户：登录后引导绑定，可「暂不绑定」直接进入，后续在「我的 - 绑定手机号」补绑。
> - 绑定 / 换绑均只需「（新）手机号 + 验证码」，**不要求旧手机号双重校验**（产品无密码凭证，换绑常因旧号停用；登录态本身即归属凭证）。
>
> **E. 手机号绑定冲突（不做账号合并）**
> - 待绑手机号在本机构已被其他账号 / 微信绑定 → 提示「该手机号已被其他账号绑定，请更换其他手机号，如有疑问请联系平台客服」，**不合并、不顶替**，引导更换手机号或联系客服。**【0613-2】占用提示页「联系平台客服」复用「我的 - 联系客服」同款弹窗（含客服二维码 / 电话 / 邮箱），不另行绘制。**
> - A 微信、B 微信先后绑定同一手机号：后者同样命中占用提示（不合并）。
> - 待绑手机号在本机构唯一 → 正常绑定成功。
>
> **F. 敏感操作手机号绑定门槛（合规）**
> - 微信登录用户可不绑手机号使用基础功能；但**支付下单、兑换码核销、开通 / 充值会员**三类操作前置校验是否已绑手机号，未绑则弹层引导前往「我的 - 绑定手机号」，完成后继续。不强制全体绑定，仅在交易 / 权益变更前要求。

### 2. AI 会话

| 功能点 | 功能描述 |
|---|---|
| Agent 形象 sticky 区 | 会话顶部固定 Agent 形象，会话区下滚渐隐隐藏、上滚复原显示。 |
| Agent 路由（一次性判定） | 扫码或 H5 链接均带 KP ID；首次进入会话按 KP ID 拉绑定 Agent 渲染形象 / 声音，并在 slogan「有问题，就问 AI 问书」下方展示系统基于 KP 自动提炼的 3 条入门示例问题（点击即发送）；后续提问不再重判定。 |
| 输入框能力开关（深度思考 / 智能搜索） | 会话输入框下方提供两个相互独立的能力开关：**深度思考**——开启后调用更强推理、给出更细致的思考型回答，**默认开启**；**智能搜索**——开启后允许联网检索补充知识库未覆盖的信息并标注来源，**受机构「联网配置」总开关约束**（该总开关由平台超管在「机构详情 / 机构配置 / 联网配置」中设定，机构未开启联网时此开关不生效），**默认关闭**。两开关均可单独切换，状态随会话保留。视觉状态：**未选中＝白底 + 黑字 + 黑 icon；选中＝主题色（靛蓝软底 + 靛蓝字 / icon）**。 |
| 流式答案 + 多模态卡片 | 文字打字机式流式输出（**仅新生成的回答展示打字机动效；切换历史会话、实时电话挂断返回等场景下，已生成的回答直接整段展示、不重复动效**）；卡片内**文字区在上、媒体区（图 / 音 / 视）在文字区下方紧凑排布**，两区做明显视觉区分；媒体并列排布支持左右滑动，**顺序不固定，按后台返回的相关度从左到右排列**（非图→音→视固定序）；受限内容按权益打锁标；点击任一媒体可放大。 |
| 答案操作（复制 / 点赞 / 反馈） | 每条 AI 答案下方、「参考 N 篇知识」入口左侧并排展示三个操作 icon：复制（复制答案文本）、点赞、反馈（点踩 / 提交问题反馈）。 **【0610】点赞 / 反馈 icon 改灰色系描边填充样式（选中态为灰色填充、非主题色），参考图 1。** |
| 输入框语音输入 | 麦克风按钮：**单击 / 长按进入语音输入态**——在**当前会话页底部弹出音浪动画浮层**（非全屏新页；浮层背底渐变与会话背景自然过渡、不割裂）；点击音浪外任意位置即关闭并发送本次语音问题、触发 AI 回答。**【0610】新增。** |
| 回答溯源富信息 | 答案下展示「参考 xx 篇知识」入口，点击后从右侧抽屉拉出溯源列表，**仅按相关度从高到低排序、不展示相关度数值（排序规则仅研发依据，界面不显示文案）**。**分两块展示，中间用浅灰横线分隔**：上块为知识 KP（每条含 KP 封面图 48×60 + KP 名称 + 文件名 + 章节页码；**同一 KP 内命中多个文件时归并在该 KP 下逐条列出，最多展示 2 个，其余以「…等 N 个文件」省略**），下块为互联网检索内容（站点用**小圆形图标**表示，不再用书本矩形封面）。字段过长尾部省略号截断。条目点击行为：book 类型跳转该 KP 的纸书购买链接，链接为空则提示「**暂不可查看详情**」（3 秒后自动消失，提示居屏幕中上方）；web 类型直接打开来源网页。 **【0613】web 类型来源跳转第三方前增加 3 秒倒计时过渡浮层「即将跳转至第三方页面…」，倒计时结束再打开来源网页。** **【0613-2】倒计时浮层扩展到所有「跳出 AI 问书」的链接——web 互联网来源 + 知识 KP 的纸书 / 运营填写链接均先弹 3 秒过渡浮层；浮层不再展示具体 URL 地址（仅文案提示）。** |
| 媒体资源区与付费墙时机 | 答案卡片内「**相关媒体资源**」作为加粗小标题直接贴在卡片上（不再单独灰底框），不展示「按相关度排序」文案。付费墙两种触发时机：① 在会话内**直接点击需付费的媒体资源** → 直接弹出付费墙（不进放大预览）；② 点击免费 / 已付费资源进入放大预览后**翻页浏览到需付费内容** → 在放大预览界面**下方**弹出付费墙（盖在预览之上，不被遮挡）。付费墙去除左侧锁图标，标题 / 描述 / 按钮整体居中。 |
| 推荐追问 | 展示在多模态卡片下方，2–3 个追问**上下纵向堆叠（非左右并列）**，点击即发送。 |
| 多模态全屏 lightbox + 版权保护 | 放大后顶部固定「因版权限制，暂不支持保存下载」；底部缩略图条可点击切换，主预览区支持左右滑动浏览同卡片其他媒体。 |
| 实时电话式语音 | **会员专属能力**：未开通会员点击电话入口时引导开通；开通后方可进入。独立电话页：中央 Agent 形象，下方居中「挂断」（返回原会话并定位最近一轮），左侧「静音」（关闭时不收音），右侧「字幕」（开启时显示字幕、Agent 形象隐藏，关闭时反之）。挂断按钮上方展示实时语音状态文案 + 动画（如「正在听…」「正在回答…」）。字幕开启时对话文本按交互时序自上而下、左对齐排列，用户文本浅灰色、AI 回复深黑色。全程问答同步写入 AI 会话历史。 |

### 3. 付费墙与权益

| 功能点 | 功能描述 |
|---|---|
| AI 会员订阅页 | 仅展示首月折扣价、月度价两档，无年付；含自动续费协议入口与 72 小时宽限期说明。 **【0613】开通 / 充值会员前校验手机号绑定（见「敏感操作手机号绑定门槛」）。** |
| 永享购买 | 触发受限内容时弹出永享购买卡，展示对应图 / 音 / 视单价与权益说明（永久解锁该单条内容）。 **【0613】支付下单前校验手机号绑定。** |
| 兑换码兑换 | 「我的 - 兑换码」入口核销会员权益，支持手动键入与调起相机 / 相册 OCR 拍照识别，需获取摄像头 / 相册权限。 **【0613】核销前校验手机号绑定。** |
| 微信支付过场 + 结果页 | **会员开通与永享买断统一走模拟微信支付收银台**（**标题区白底、与下方区域一致；AI 问书 logo 用会话同款电光靛紫色动画球**；商户、金额、零钱支付方式、确认支付按钮沿用微信绿、模拟环境提示）→ 确认 → 成功结果页。成功页为品牌质感设计（**支付成功对勾用绿色 / 主题色 + 同心圆光环，替代原珊瑚 / 朝霞橙，避免被误解为未支付成功**），「返回会话」**定位到支付前掉起支付的那个会话**（非新会话）。 |

### 4. 我的

| 功能点 | 功能描述 |
|---|---|
| 账户管理 | 仅保留「换绑手机号」（新号 + 验证码校验）。 **【0613】确认换绑仅需「新手机号 + 验证码」，不要求旧手机号双重校验**（产品无密码凭证、换绑多因旧号停用；登录态本身即归属凭证，异常场景留待生产期风控 / 客服申诉）。 |

>

## 附：数据看板指标表（机构后台）

> 整个看板顶部统一时间区间筛选（今日 / 7 日 / 30 日 / 自定义），全 Tab 与图表联动；整页支持导出。

### 总览
| 指标 | 公式 | 业务意义 |
|---|---|---|
| 累计注册用户 | COUNT(DISTINCT user_id) | 规模 |
| 当前会员数 | 会员未到期的去重用户数 | 商业化基本盘 |
| 累计 GMV | SUM(订单金额 WHERE 状态=已支付) | 总营收 |
| 当日 DAU | COUNT(DISTINCT user_id WHERE 当日有登录或发问) | 即时活跃度 |
| 累计退款金额 / 退款率 | SUM(退款成功金额)；退款率 = 退款额 ÷ GMV | 售后与资金健康度（**【0613】新增**） |

### 用户分析
| 指标 | 公式 | 业务意义 |
|---|---|---|
| DAU / WAU / MAU | 周期内有效行为去重用户数 | 粘性分层 |
| 新增用户数 | 当日首次注册数 | 拉新效果 |
| 次日 / 7 日 / 30 日留存率 | 新增用户在 N 日后仍活跃占比 | 产品价值是否站得住 |
| 来源分布 | 扫码进入 vs 直接访问 | 渠道效果 |
| 活跃时段分布 | 按小时统计提问数 | 运营 / 推送时点参考 |
| 地区分布 / 性别分布 | 按地区、性别分组的用户占比 | 用户画像与精细化运营（**【0613】新增**） |

### 提问分析
| 指标 | 公式 | 业务意义 |
|---|---|---|
| 总提问数 | COUNT(提问) | 使用强度 |
| 人均提问数 | 总提问数 / DAU | 单用户深度 |
| 平均会话轮次 | 总提问数 / 总会话数 | 对话沉浸度 |
| 受限内容触发率 | 触发付费墙次数 / 总提问数 | 付费墙强度是否合理 |
| 按 Agent 提问分布 | 每个 Agent 接收提问占比 | Agent 配置 / 路由质量 |
| 提问领域分布 | 按 KP / 领域聚合的提问占比 | 用户最关心的领域（**【0613】新增**） |
| 答案点赞率 / 反馈率 | 点赞数 ÷ 答案数；反馈（点踩）数 ÷ 答案数 | 答案质量与知识库迭代（**【0613】新增**） |

### 转化分析
| 指标 | 公式 | 业务意义 |
|---|---|---|
| 付费用户数 / 付费转化率 | 付费用户 / 累计用户 | 商业化核心 |
| 会员漏斗 | 看到会员页 → 点击购买 → 完成支付 三步转化 | 找漏斗瓶颈 |
| 永享转化率 | 购买永享数 / 触发永享付费墙数 | 永享定价是否合理 |
| ARPPU | 总收入 / 付费用户数 | 单付费用户产值 |
| 续费率 | 到期续费数 / 到期会员数 | 长期价值 |

### 热门 KP
| 指标 | 取数 | 业务意义 |
|---|---|---|
| 提问数 TOP 10 KP | 按 KP 维度统计提问 | 真正被使用的内容 |
| 付费转化贡献 TOP 10 KP | 通过该 KP 触发的会员 / 永享订单数 | 营销价值高的内容 |
| 永享购买 TOP 10 KP | 按 KP 永享订单数 | 永享潜力 |
