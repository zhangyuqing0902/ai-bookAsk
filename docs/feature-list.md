# AI 问书 · 三端功能清单（开发依据 / 测试标准）

> 版本：v2 重建 · 日期：2026-06-16
> 范围：移动端 H5 / 机构后台 PC / 平台超管后台 PC，三端高保真原型（只 mock，不接真后端）
> 组织：端 → 模块 → 功能点（一级）→ 二级功能；功能描述以（1）（2）（3）有序编号逐条左对齐排列，有层级时用（1.1）（1.2）
> 口径：以产品生产意图描述；〔演示〕＝仅原型演示用、非上线功能，〔评审〕＝评审辅助工具
> 注：本清单严格对照 apps/ 现有代码逐页核对，与代码一致。

---

## 端一 · 移动端 H5（C 端用户）

### 1. 登录与授权

#### 1.1 登录落地页

（1）页面仅展示 AI 问书 知识核动效 + 标题 + slogan「答案有出处 · 知识更可信」+ 底部「知识由 中国医学临床百家 提供」，不展示「你好我是…AI 版作者」问候语
（2）两个登录入口：「微信授权登录」（绿色）、「手机号登录」（主题色）
（3）协议勾选区：默认未勾选，点整行切换；含《用户协议》《隐私政策》两链接（点链接不触发勾选）
（4）校验：未勾选协议点任一登录按钮 → toast「请先阅读并同意《用户协议》与《隐私政策》」，不跳转
（5）微信登录按打开环境分流：微信内 → 官方授权弹窗页；外部浏览器 → 微信扫码授权页（由 UserAgent 是否含 MicroMessenger 自动判定）
（6）手机号登录 → 手机号验证码登录页
（7）〔演示〕协议下方有「仅用于演示 · 环境切换（非上线功能）」区块：① 微信内/浏览器 切换（决定授权与支付走哪条通道）② Token 正常/机构 Token 超限 切换（影响会话页提示与拦截）；生产环境无此切换

#### 1.2 微信授权登录（微信内场景）

（1）微信 App 内打开时点「微信授权登录」不出二维码，直接唤起微信官方授权弹窗
（2）授权弹窗：AI 问书 logo + 「申请使用」+ 权限项「获取你的昵称、头像、性别与地区」
（3）点「允许」→ 携 code 走授权回调，自动带回 昵称/头像/性别/地区 写入账户 → 进入 AI 会话首页（手机号为空，后续按需绑定）
（4）点「拒绝」→ 留在登录落地页
（5）H5 网页授权（snsapi_userinfo）无法获取手机号（一键授权手机号为小程序专属），故不弹「获取手机号」授权窗，手机号统一改由用户手动「手机号+验证码」绑定

#### 1.3 微信扫码登录（浏览器场景）

（1）非微信浏览器点「微信授权登录」→ 跳微信扫码授权页，由微信域名 open.weixin.qq.com 渲染二维码
（2）二维码采用内嵌样式（WxLogin JS SDK，整页可美化），二维码中央不放 AI 问书 Logo（二维码属微信，避免误导）
（3）用户微信扫码 → 手机端确认「允许/拒绝」；允许携 code 回调，流程同微信内 → 进入首页
（4）「返回其他登录方式」回落地页
（5）〔演示〕扫码-已扫描-已确认 三阶段自动播放动画，模拟扫码登录全过程

#### 1.4 手机号验证码登录

（1）手机号输入框（限 11 位数字，自动过滤非数字）+ 验证码输入框（限 6 位）+「获取验证码」（点击 toast「验证码已发送」）
（2）登录按钮在手机号或验证码为空时禁用
（3）校验：手机号非 11 位 → toast「请输入正确的 11 位手机号」；验证码非 6 位 → toast「请输入 6 位验证码」
（4）校验通过 → 登录成功进入 AI 会话页；此方式不获取、不绑定任何微信信息

#### 1.5 绑定手机号

（1）三种来源差异化副标题：登录后引导（绑定后可跨端同步会话与权益）/ 敏感操作门槛（支付下单、兑换码与开通会员需先完成手机号绑定）/ 我的入口（绑定后可使用支付、兑换码与会员等权益）
（2）手机号 + 验证码两输入框（校验同手机号登录）+「获取验证码」
（3）主按钮文案随来源变（绑定并进入 / 完成绑定）；空值禁用
（4）绑定成功 → toast「手机号绑定成功」→ 按来源跳转（登录来源进会话页，其余返回上一页）
（5）绑定/换绑仅需「（新）手机号 + 验证码」，不要求旧手机号双重校验（产品无密码凭证，登录态本身即归属凭证）
（6）登录来源额外提供「暂不绑定 · 直接进入」可跳过；所有来源都有「手机号已被占用?」入口

#### 1.6 手机号被占用提示

（1）待绑手机号在本机构已被其他账号/微信绑定 → 提示「该手机号已被其他账号绑定，请更换其他手机号，如有疑问请联系客服」
（2）不做账号合并、不顶替（A、B 两微信先后绑同一手机号同样命中占用提示）
（3）按钮：「更换手机号」（返回绑定页）、「联系客服」（复用「我的-联系客服」同款弹窗，含客服二维码/电话/邮箱）

### 2. AI 会话

#### 2.1 会话主界面

（1）顶栏：左侧菜单（开历史会话抽屉）、中间「AI 问书 / 内容由 AI 生成」、右侧实时电话入口 + 新建会话
（2）会话内容跨页保留（跳会员/支付/电话页再返回仍是同一会话）
（3）欢迎态（无消息）：浮动知识核 + slogan「有问题，就问 AI 问书」+ 系统基于 KP 自动提炼的 3 条入门示例问题（点击即发送）
（4）对话态：用户气泡 + AI 答案卡片
（5）输入框：placeholder「发消息或点击右侧说话…」，Enter 发送；有文字显发送按钮、无文字显语音按钮
（6）Agent 与检索分离（3.4）：保留每 KP 专属 Agent——扫码/链接带 KP ID，首次进入按 KP ID 拉绑定 Agent 渲染形象/声音/引导问题；回答检索范围为当前机构全部 KP（不局限进入的单个 KP），答案标注来源 KP
（7）欢迎态透明说明（3.4）：Agent 形象下方一行淡靛底小字「综合「{机构名}」全部知识为你解答」，告知答案来自全机构知识库；仅欢迎态首次展示一次，发消息后消失

**二级功能 — 能力开关（深度思考 / 智能搜索）**

（1）输入框下方两个相互独立的能力开关，状态随会话保留
（2）深度思考：开启后调用更强推理、给更细致的思考型回答，默认开启
（3）智能搜索：开启后允许联网检索补充知识库未覆盖信息并标注来源，默认关闭
（4）智能搜索受机构「联网配置」总开关约束（平台超管在机构详情设定，机构未开启联网时此开关不生效）
（5）视觉：未选中＝白底黑字黑 icon；选中＝主题色（靛蓝软底 + 靛蓝字/icon）

#### 2.2 流式问答（打字机）

（1）AI 答案文字打字机式流式输出，未输出完时显示光标
（2）仅新生成的回答展示打字机动效；切换历史会话、电话挂断返回等场景下已生成回答直接整段展示、不重复动效

#### 2.3 相关媒体资源（多模态卡片）

（1）答案卡片内「相关媒体资源」作为加粗小标题贴在卡片上；文字区在上、媒体区在下，两区做明显视觉区分
（2）媒体并列排布支持左右滑动，顺序按后台返回的相关度从左到右排列（非图→音→视固定序），不展示相关度数值
（3）受限内容在封面上展示权益标签 + 类型角标（图片/音频/视频）：会员内容标「会员」、永享内容标「永享」，便于用户一眼区分付费类型
（4）一个资源若既是会员又设了永享价，按权益边界归永享池——只标「永享」、只走永享购买
（5）付费墙两种触发时机：① 直接点击需付费媒体 → 直接弹付费墙（不进预览）② 点免费/已付费进放大预览后翻页到需付费内容 → 在预览下方弹付费墙

#### 2.4 答案操作与反馈

（1）每条 AI 答案下方、「参考 N 篇知识」入口左侧并排三个 icon：复制（toast「已复制答案」）、点赞、点踩
（2）点赞/点踩互斥，可再次点击取消；icon 为灰色系描边填充样式（选中态灰填充、非主题色）

**二级功能 — 点踩反馈弹层**

（1）点踩 → 弹底部反馈 sheet，标题「反馈」
（2）4 个互选标签：有害/不安全、虚假信息、没有帮助、其他
（3）可填写原因文本框（placeholder「我们想知道你对此回答不满意的原因…」）
（4）「提交」→ 该答案标记点踩 + toast「已记录反馈,感谢你的帮助」

#### 2.5 推荐追问

（1）展示在多模态卡片下方，2–3 个追问上下纵向堆叠（非左右并列）
（2）点击任一即作为新问题发送

#### 2.6 语音输入（短语音）

（1）点输入框右侧麦克风 → 在当前会话页底部弹音浪动画浮层（非全屏新页），浮层背底与会话背景自然过渡
（2）提示「松手发送，上滑取消」
（3）点浮层外任意位置即关闭并发送本次语音问题、触发 AI 回答

#### 2.7 历史会话抽屉

（1）左上角菜单开抽屉；顶部「新建会话」
（2）历史按时间分组（今天/昨天/某年某月）列出，点任一条载入该会话
（3）页脚用户卡（头像+昵称+会员标+手机号）点击进入「我的」

#### 2.8 答案溯源抽屉

（1）答案下「参考 N 篇知识」入口，点击从右侧抽屉拉出溯源列表，仅按相关度从高到低排序、不显数值
（2）分两块、中间浅灰横线分隔：上块知识 KP（KP 封面 + KP 名 + 文件名 + 章节页码；同 KP 多命中文件归并最多 2 个、其余「…等 N 个文件」），下块互联网检索（站点用小圆形图标）
（3）字段过长尾部省略号截断
（4）条目点击：book 类型跳该 KP 纸书购买链接，链接为空提示「暂不可查看详情」；web 类型打开来源网页

#### 2.9 外部来源跳转倒计时

（1）所有「跳出 AI 问书」的链接（web 互联网来源 + 知识 KP 的纸书/运营填写链接）均先弹过渡提示
（2）样式为屏幕中上方黑底白字倒计时条：左侧 3·2·1 倒计时 + 「即将跳转到外部」+ 右侧「取消跳转」
（3）倒计时结束打开来源网页；点「取消跳转」中止；触发时不关闭右侧溯源抽屉，提示叠在当前页之上
（4）倒计时浮层不展示具体 URL（仅文案提示）

#### 2.10 机构 Token 超限拦截

（1）〔演示〕由落地页演示开关触发；生产由机构实际 Token 用量判定
（2）超限时会话顶部常驻珊瑚红提示条「当前机构 Token 已达上限，请联系客服」
（3）发送消息、语音输入、点实时电话入口均被拦截；被拦时让顶部提示条放大抖动+跳亮提醒，不再额外弹 toast

#### 2.11 实时电话式语音

（1）会员专属：未开通会员点电话入口时引导开通；开通后方可进入
（2）独立电话页：中央 Agent 形象 + 实时语音状态文案与动画（如「正在听…」「正在回答…」）
（3）底部三键：静音（关闭时不收音）、挂断（返回原会话并定位最近一轮）、字幕（开启显字幕、Agent 形象隐藏）
（4）字幕开启时对话文本按时序自上而下左对齐，用户文本浅灰、AI 回复深黑
（5）全程问答同步写入 AI 会话历史

### 3. 付费墙与权益

#### 3.1 多模态付费墙

（1）受限媒体触发，按资源所属权益池动态呈现按钮（不再固定两按钮）：
（2）会员资源：标题「{资源名} · 会员内容」+ 单按钮「开通会员 · 畅享全部会员内容」（进会员页）
（3）永享资源：标题「{资源名} · 永享内容」+ 单按钮「¥X 永久解锁此内容」（经手机号绑定门槛后进微信支付收银台），文案点明「永享内容单独购买，无需开通会员」
（4）金额取该资源的永享价（非硬编码）；付费墙去除左侧锁图标，标题/描述/按钮整体居中

#### 3.2 多模态 lightbox 预览

（1）放大预览顶部固定「因版权限制，暂不支持保存下载」+ 关闭✕
（2）支持左右翻页（循环）+ 底部缩略点/缩略图条切换，主预览区可左右滑动浏览同卡片其他媒体
（3）图片可缩放（百分比显示）；视频控件紧贴视频下方，支持播放/暂停、±15s 快进快退、进度条拖动、横竖屏切换；音频显音浪 + 进度条 + 时间码

#### 3.3 开通会员

（1）仅展示首月折扣价、月度价两档，无年付；首月特惠默认选中
（2）含自动续费协议入口与「可随时退订 · 到期前 72h 宽限期」说明
（3）「立即开通」前校验手机号绑定（未绑先引导绑定再继续），通过后按所选档进微信支付收银台

#### 3.4 会员中心

（1）会员卡：会员标识 + 有效期 + 续费状态（自动续费 / 到期不再续费）
（2）「免费 vs 会员」权益对比表（逐项 ✅/✗）：基础文字知识问答、图音视频深度知识精讲、VIP 极速优先通道、实时电话即时问答、永享名家典藏知识（永享标注为「单独购买」）
（3）底部说明「自动续费将于到期前 3 天短信提醒，可随时取消」

**二级功能 — 取消/开启自动续费**

（1）自动续费开启时显示「取消自动续费」入口（无手动续费按钮）
（2）点取消 → 二次确认 sheet（挽留设计：主按钮「暂不取消」、弱化「确认取消」）
（3）确认取消 → toast「已取消成功,会员有效期至 X 后将不再自动续费」
（4）重新开启自动续费（经手机号绑定门槛）→ toast「已开启自动续费 · 将于到期前 3 天短信提醒」

### 4. 我的

#### 4.1 我的主页

（1）顶部用户卡（点击进个人资料）：头像 + 昵称（无则「未设置昵称」）+ 会员标 + 手机号（未绑显「未绑定手机号」）
（2）「会员」分组：会员中心（右值 已开通/未开通）
（3）「我的」分组：我的永享、我的纸书、我的订单、兑换码
（4）「账户与帮助」分组：换绑/绑定手机号（未绑显红字「未绑定」）、联系客服、用户协议、隐私政策
（5）退出登录 → toast「已退出登录」→ 回落地页
（6）〔评审〕「评审工具」分组：原型清单（三端站点地图）

#### 4.2 我的永享

（1）已购永享列表，按文件名模糊搜索 + 类型筛选（全部/图片/音频/视频）
（2）网格卡片：封面 + 类型角标 + 名称 +「{类型} · 永久解锁」
（3）点卡片进多模态预览（顶部固定版权保护文案，不支持保存下载）
（4）空态「没有匹配的永享内容」

#### 4.3 我的纸书

（1）展示用户所有线下扫码对应的知识 KP（不只首扫绑定的），含已解锁与未解锁
（2）卡片：KP 封面 + KP 名（固定两行高）+「扫码解锁 · 时间」（钉在卡片底部对齐）；按 KP 名模糊搜索
（3）状态区分：顶部筛选 chip（全部 / 已解锁 / 未解锁，与「我的永享」同款 UI，默认「全部」）；首扫绑定的 KP 卡片显「已解锁」标，后扫未获权益不显状态标（界面保持干净）；列表说明文案「展示全部扫过纸书，首扫解锁相应权益」
（4）首扫绑定送「该 KP 会员级权益」：可看这本书的免费 + 会员内容；该 KP 的永享内容仍需单独购买
（5）无论是否已解锁，点卡片都直接进入该 KP 会话；区别在内容层——已解锁可直接看该 KP 会员内容，未解锁点会员内容需按该 KP 基础权益（免费/会员）付费；永享内容无论是否解锁都需单独永享付费
（6）进入会话仍弹 3 秒黑底 toast「即将进入「{KP名}」AI 会话」（文案过长自动换行）

**二级功能 — 扫一扫**

（1）右上角「扫一扫」入口，点击调起手机相机扫码
（2）扫到纸书二维码后 toast「已识别纸书二维码 · 正在进入全新会话」→ 进入对应 KP 的全新会话并绑定该 KP 权益

#### 4.4 个人资料

（1）由「我的」顶部头像/昵称/手机号区右侧箭头进入（不再有独立「个人资料」菜单）

## 端四 · 三端通用（全局交互）

### 1. 通用状态规范

#### 1.1 Toast 轻提示

（1）单条、统一位置弹出（移动端底部 / 后台顶部），默认数秒自动消失
（2）用于操作结果反馈（已保存/已删除/已复制等），文案过长自动换行不超屏

#### 1.2 二次确认弹窗（ConfirmDialog）

（1）危险/不可逆操作（删除/下架/停用/退款/取消分享等）统一走二次确认
（2）危险操作主按钮为 danger 红色，文案写清后果与是否可恢复

#### 1.3 加载骨架屏

（1）后台列表/详情挂载先显骨架屏占位，再渲染真实数据，避免白屏跳变

#### 1.4 空态

（1）列表/搜索/筛选无结果时显居中插画 + 标题 + 副文案（如「没有匹配的 X / 换个条件试试」）
（2）部分空态内嵌主操作入口（如「新建 X」）

#### 1.5 分页

（1）后台列表默认每页 10 条，超过才分页；筛选/搜索变更后回到第 1 页；分页单位随实体变化（家/个/笔/条/单等）

#### 1.6 操作列固定

（1）后台表格「操作」列固定在右侧（白底+左侧分隔线+阴影），横向滚动时常驻可见

#### 1.7 数值单位规范

（1）两后台数值统一中文进制缩写：<1 万千分位；1 万~1 亿 → X.X 万；≥1 亿 → X.XX 亿；Token 统一以「亿」为单位
（2）各看板页脚附统一单位规范说明

#### 1.8 登录态有效期

（1）移动端登录态有效期 30 天，每次操作自动重置有效期
